🍽️ Sistema de Pedidos Online – Versão Simples para Apresentação

🎯 Objetivo:
Permitir que um cliente selecione pastéis, envie um pedido, e o administrador veja e atualize o status desse pedido.

✅ Como ele cumpre os requisitos?

Requisito                         | Como será implementado
----------------------------------|-------------------------
CRUD completo                     | Em duas entidades: Pastéis e Pedidos
Relacionamento Um para Muitos     | Um pedido tem muitos itens de pedido (pastéis)
Login com autenticação fake       | Área administrativa com login PHP + SESSION
Evento JavaScript                 | Alerta confirm() ao remover um item do carrinho
CSS                               | Estilização básica com CSS ou Bootstrap
Código modularizado               | Separar arquivos: login, cadastro, páginas, includes, etc

🧱 Estrutura Simples do Banco de Dados (MySQL)

CREATE DATABASE pastelaria_fidelim
DEFAULT CHARACTER SET utf8mb4
DEFAULT COLLATE utf8mb4_unicode_ci;

USE pastelaria_fidelim;

CREATE TABLE usuarios (
    id INT AUTO_INCREMENT PRIMARY KEY,
    login VARCHAR(50),
    senha VARCHAR(50)
);

CREATE TABLE pasteis (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nome VARCHAR(100),
    preco DECIMAL(5,2),
    imagem VARCHAR(255)
);

CREATE TABLE pedidos (
    id INT AUTO_INCREMENT PRIMARY KEY,
    cliente_nome VARCHAR(100),
    status ENUM('Em preparo', 'Pronto', 'Entregue') DEFAULT 'Em preparo',
    data_pedido DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE itens_pedido (
    id INT AUTO_INCREMENT PRIMARY KEY,
    id_pedido INT,
    id_pastel INT,
    quantidade INT,
    FOREIGN KEY (id_pedido) REFERENCES pedidos(id),
    FOREIGN KEY (id_pastel) REFERENCES pasteis(id)
);

🖼️ Telas do Sistema (Versão Simples)

🔓 1. Tela de Login (admin)
- Login e senha fixos (ex: admin / 1234)
- Protege a área administrativa

🛒 2. Tela do Cardápio (cliente)
- Lista os pastéis com nome, preço e imagem
- Botão "Adicionar ao Carrinho"

🧺 3. Carrinho de Compras
- Lista de itens adicionados
- Botão "Finalizar Pedido"
- Formulário com nome do cliente

👨‍🍳 4. Painel Administrativo
- Lista todos os pedidos com:
  - Nome do cliente
  - Data
  - Lista dos pastéis
  - Status (com botão para alterar)
- Ações: mudar status (Em preparo → Pronto → Entregue)

🧠 Fluxo Resumido do Sistema
1. Cliente acessa a página principal e vê o cardápio.
2. Adiciona pastéis ao carrinho.
3. Preenche o nome e envia o pedido.
4. Pedido é salvo no banco com status "Em preparo".
5. Admin faz login, acessa o painel, vê os pedidos.
6. Admin altera o status conforme o preparo.
