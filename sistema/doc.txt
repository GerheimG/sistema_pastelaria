 Sistema de Pedidos Online – Pastelaria Fidelim (Versão Simples)
 Objetivo

Desenvolver um sistema online simples para pedidos de pastéis, permitindo que o cliente visualize o cardápio, monte um carrinho de compras e envie o pedido. Um administrador pode visualizar os pedidos e atualizar seus status conforme o preparo.

Requisitos Atendidos
Requisito	Implementação
CRUD completo	CRUD realizado para entidades Pastéis e Pedidos
Relacionamento Um para Muitos	Um pedido contém vários itens de pedido (pastéis)
Login com autenticação (simulada)	Login do administrador com verificação via PHP + SESSION
Evento JavaScript	Uso do confirm() ao remover itens do carrinho
Estilização com CSS	Aplicação de CSS ou Bootstrap para layout e responsividade
Código modularizado	Separação em múltiplos arquivos: login, includes, ações, etc.
 Estrutura do Banco de Dados (MySQL)
CREATE DATABASE pastelaria_fidelim
DEFAULT CHARACTER SET utf8mb4
DEFAULT COLLATE utf8mb4_unicode_ci;

USE pastelaria_fidelim;

-- Tabela de Usuários (Admin)
CREATE TABLE usuarios (
    id INT AUTO_INCREMENT PRIMARY KEY,
    login VARCHAR(50),
    senha VARCHAR(50)
);

-- Tabela de Pastéis
CREATE TABLE pasteis (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nome VARCHAR(100),
    preco DECIMAL(5,2),
    imagem VARCHAR(255)
);

-- Tabela de Pedidos
CREATE TABLE pedidos (
    id INT AUTO_INCREMENT PRIMARY KEY,
    cliente_nome VARCHAR(100),
    status ENUM('Em preparo', 'Pronto', 'Entregue') DEFAULT 'Em preparo',
    data_pedido DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- Tabela de Itens do Pedido
CREATE TABLE itens_pedido (
    id INT AUTO_INCREMENT PRIMARY KEY,
    id_pedido INT,
    id_pastel INT,
    quantidade INT,
    FOREIGN KEY (id_pedido) REFERENCES pedidos(id),
    FOREIGN KEY (id_pastel) REFERENCES pasteis(id)
);

Telas do Sistema
1. Tela de Login (Admin)

Acesso restrito ao painel administrativo

Autenticação simples com login fixo (ex: admin / 1234)

Protegido por SESSION em PHP

🍽️ 2. Tela de Cardápio (Cliente)

Exibe lista de pastéis com:

Nome

Preço

Imagem

Botão “Adicionar ao Carrinho”

🛒 3. Carrinho de Compras

Exibe os itens adicionados

Botão para remover item (com confirmação via JavaScript)

Campo para nome do cliente

Botão “Finalizar Pedido”

4. Painel Administrativo

Listagem de todos os pedidos:

Nome do cliente

Data do pedido

Itens solicitados

Status atual

Botões de ação para alterar o status:

"Em preparo" → "Pronto" → "Entregue"

Fluxo Resumido do Sistema
graph TD
    A[Cliente acessa o cardápio] --> B[Adiciona pastéis ao carrinho]
    B --> C[Preenche nome e finaliza pedido]
    C --> D[Pedido salvo no banco com status "Em preparo"]
    D --> E[Admin faz login no painel]
    E --> F[Admin visualiza e altera o status dos pedidos]

Observações Finais

A proposta é simples, mas cobre todos os principais conceitos: sessões, banco de dados relacional, CRUD, MVC básico, modularização, e interação cliente-servidor.

O projeto pode ser facilmente expandido para incluir:

Cadastro de novos pastéis

Histórico de pedidos por cliente

Relatórios e filtros no painel administrativo

Integração com APIs de pagamento ou delivery
